# 면접 준비 — Kong AI Gateway 복구 가이드

> 이 파일은 클로드 세션이 종료되어도 복구 방법을 확인할 수 있도록 별도 보관합니다.

---

## 면접 당일 순서

### 1단계 — 게이트웨이 시작 (면접 30분 전)

```powershell
cd C:\workspaces\taesikpjt\kong-ai-gateway\deploy\scripts
python gateway-start.py
```

약 3~5분 후 아래와 같이 출력됩니다.

```
=======================================================
  GATEWAY HEALTHY!
=======================================================
  Kong URL  : http://<NEW_IP>:8000
  Admin URL : http://<NEW_IP>:8001

── Claude Code 연결 설정 ──────────────────────────────
  [영구 설정 - Windows 레지스트리]
  [Environment]::SetEnvironmentVariable(
    "AWS_ENDPOINT_URL", "http://<NEW_IP>:8000", "User")
  ※ VS Code를 재시작해야 환경변수가 반영됩니다.
```

### 2단계 — Claude Code 연결

출력된 IP를 복사해서 PowerShell에서 실행합니다.

```powershell
# <NEW_IP> 자리에 출력된 IP를 입력
[System.Environment]::SetEnvironmentVariable("AWS_ENDPOINT_URL", "http://<NEW_IP>:8000", "User")
```

그 다음 **VS Code 완전 종료 후 재시작**합니다.

> ⚠ ECS는 시작할 때마다 IP가 바뀔 수 있습니다. 반드시 출력된 새 IP를 사용하세요.

### 3단계 — 동작 확인

VS Code 재시작 후 Claude Code에서 간단한 질문으로 테스트합니다.

```
"안녕하세요, 오늘 날씨 알려주세요"
```

Kong → Guardrails → Bedrock 경로로 정상 응답이 오면 준비 완료입니다.

## 문제 발생 시

**gateway-start.py 타임아웃 시** — CloudWatch Logs 확인:
- `/ecs/kong-ai-gateway/guardrails`
- `/ecs/kong-ai-gateway/kong`

**Claude Code 연결 안 될 때** — 체크리스트:
1. `AWS_ENDPOINT_URL` 환경변수가 새 IP로 설정되어 있는가?
2. VS Code를 완전히 재시작했는가?
3. ECS 태스크가 HEALTHY 상태인가? (`gateway-start.py` 출력 확인)

**ECS 상태 수동 확인:**
```powershell
python -c "
import os, boto3
os.environ.pop('AWS_ENDPOINT_URL', None)
ecs = boto3.client('ecs', region_name='us-east-1')
svc = ecs.describe_services(cluster='kong-ai-gateway-cluster', services=['kong-ai-gateway-service'])['services'][0]
print('desired:', svc['desiredCount'], '/ running:', svc['runningCount'])
"
```
